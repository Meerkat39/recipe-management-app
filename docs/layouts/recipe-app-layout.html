ホーム レシピ一覧 チキンカレー

チキンカレー

⏱️ 調理時間: 45分 👥 4人分 📅 作成日: 2025/07/25 👀 閲覧数: 12回

📝 材料

じゃがいも 3個（中サイズ）

👩‍🍳 作り方

鶏もも肉を一口大に切り、玉ねぎは薄切り、にんじんとじゃがいもは乱切りにする。
鍋にサラダ油を熱し、鶏肉を中火で炒める。表面に焼き色が付いたら取り出す。
同じ鍋で玉ねぎを炒め、透明になったらにんじんとじゃがいもを加えてさらに炒める。
水を加えて煮立たせ、鶏肉を戻し入れて弱火で20分ほど煮込む。
火を止めてカレールーを割り入れ、よく溶かしてから再び弱火で5分ほど煮込んで完成。

📋 MVP機能の実装ポイント

🔍 検索・フィルタリング機能

  • レシピ名での部分一致検索
  • 材料名での部分一致検索
  • 調理時間・人数でのフィルタリング
  • リアルタイム検索結果表示
  • 検索結果のハイライト表示

📝 レシピ管理（CRUD）

  • レシピの新規作成・編集・削除
  • 材料の動的追加・削除
  • 必須項目の入力バリデーション
  • 下書き保存機能
  • 削除確認ダイアログ

🎨 UI/UX設計

  • Webアプリケーション向けレイアウト
  • サイドバーナビゲーション
  • レスポンシブデザイン対応
  • 直感的な操作性
  • 視覚的フィードバック

🚀 技術的実装要件

  • データベース設計（recipes, ingredients）
  • REST API設計
  • フロントエンド状態管理
  • 入力値のサニタイズ
  • エラーハンドリング

💡 開発のポイント

このMVP版では、基本的なレシピ管理と検索機能に集中し、ユーザーが直感的に操作できるWebアプリケーションとして設計しています。
将来的な機能拡張（画像アップロード、カテゴリ分類、栄養管理等）を見据えた拡張性のある設計を心がけています。

💻 技術スタック推奨

🎨 フロントエンド

  • React.js / Vue.js / Angular
  • CSS Framework (Tailwind CSS推奨)
  • 状態管理 (Redux/Vuex)
  • フォームライブラリ

🔧 バックエンド

  • Node.js / Python / Java
  • Express.js / Django / Spring Boot
  • REST API設計
  • バリデーションライブラリ

🗄️ データベース

  • PostgreSQL / MySQL (推奨)
  • MongoDB (NoSQL選択時)
  • レシピ・材料テーブル設計
  • 全文検索インデックス

🎉 MVP完成後の次ステップ

基本機能の実装完了後は、ユーザーフィードバックを収集し、Phase 2A（画像機能、カテゴリ機能）の開発に進みます。
段階的な機能追加により、より充実したレシピ管理システムを構築していきます。
